# Wood Place Property Management Website

## Project Overview

This is a modern, responsive static website for Wood Place Property Management, a residential property management company. The site showcases their services, properties, and contact information in a professional single-page layout.

## Tech Stack

- **Static Site Generator**: Jekyll 3.10 (via github-pages gem)
- **Styling**: Tailwind CSS 3.4
- **Build Tools**: Node.js (for Tailwind), Ruby (for Jekyll)
- **CSS Processing**: PostCSS with Autoprefixer
- **Font**: Inter (Google Fonts)
- **Deployment**: GitHub Pages (automatic building enabled)

## How It Works

### Build Pipeline

The project uses a dual build system:

1. **Tailwind CSS Compilation**:
   - Source: `assets/css/main.css` (contains Tailwind directives)
   - Output: `assets/css/styles.css` (compiled CSS)
   - Process: Tailwind scans HTML files for class names and generates the final CSS

2. **Jekyll Site Generation**:
   - Templates: `_layouts/default.html`
   - Content: `index.html` (uses Jekyll front matter)
   - Output: `_site/` directory (ready for deployment)

### Critical Build Requirement

**The CSS must be compiled before viewing the site**. The layout file references `assets/css/styles.css` which doesn't exist in source control - it's generated by the Tailwind build process.

## Development

### First Time Setup

```bash
# Install Ruby dependencies
bundle install

# Install Node.js dependencies
npm install

# Compile CSS (required before first run)
npx tailwindcss -i ./assets/css/main.css -o ./assets/css/styles.css
```

### Running Development Server

**Option 1: Run both processes together** (recommended)
```bash
npm run dev
```

**Option 2: Run separately**
```bash
# Terminal 1: Compile CSS with watch mode
npm run build:css

# Terminal 2: Run Jekyll server
npm run build:jekyll
```

The site will be available at `http://localhost:4000`.

### Making Changes

- **Content**: Edit `index.html`
- **Layout/HTML structure**: Edit `_layouts/default.html`
- **Styling**: Add Tailwind classes to HTML or custom CSS to `assets/css/main.css`
- **Colors/Theme**: Modify `tailwind.config.js`
- **Site config**: Edit `_config.yml`

## Common Issues & Troubleshooting

### Styles Not Loading

**Symptom**: Page loads but has no styling (unstyled HTML).

**Cause**: The `assets/css/styles.css` file hasn't been generated.

**Solution**: Run Tailwind compilation:
```bash
npx tailwindcss -i ./assets/css/main.css -o ./assets/css/styles.css
```

### Port Already in Use

**Symptom**: Jekyll fails to start with "port is in use" error.

**Solution**:
- Find and kill existing Jekyll process: `pkill -f jekyll`
- Or use a different port: `bundle exec jekyll serve --port 4001`

### Tailwind Classes Not Working

**Symptom**: Added new Tailwind classes but they don't appear.

**Cause**: Tailwind build process needs to run to include new classes.

**Solution**:
- If using watch mode (`npm run build:css`), save a file to trigger rebuild
- If not in watch mode, manually rebuild: `npx tailwindcss -i ./assets/css/main.css -o ./assets/css/styles.css`

### Changes Not Appearing

**Symptom**: Made changes but they don't show on the site.

**Solutions**:
- For HTML/content changes: Jekyll auto-regenerates (just refresh browser)
- For CSS/Tailwind changes: Ensure Tailwind watch is running
- Hard refresh browser: Ctrl+Shift+R (Windows/Linux) or Cmd+Shift+R (Mac)

## Production Build

### Local Build

```bash
# 1. Build optimized CSS
npx tailwindcss -i ./assets/css/main.css -o ./assets/css/styles.css --minify

# 2. Build Jekyll site
bundle exec jekyll build

# Output is in _site/ directory
```

### GitHub Pages Deployment

This site is configured to use GitHub Pages automatic building:

1. **Commit the compiled CSS**: The `assets/css/styles.css` file must be committed to the repository
   ```bash
   git add assets/css/styles.css
   git commit -m "Update compiled CSS"
   ```

2. **Push to GitHub**: Push your changes to the `main` branch
   ```bash
   git push origin main
   ```

3. **Enable GitHub Pages**:
   - Go to your repository settings on GitHub
   - Navigate to "Pages" in the sidebar
   - Under "Source", select the `main` branch
   - GitHub will automatically build and deploy your site

4. **CSS Compilation**: Since GitHub Pages builds with Jekyll only, Tailwind CSS must be compiled locally and the `styles.css` file must be committed to the repository. GitHub Pages will serve the pre-compiled CSS file.

**Important**: Unlike development, GitHub Pages does not run the Tailwind build process. Always compile and commit your CSS before pushing:
```bash
npx tailwindcss -i ./assets/css/main.css -o ./assets/css/styles.css --minify
git add assets/css/styles.css
git commit -m "Update compiled CSS"
git push
```

## Notes for AI Assistants

- Always check if `assets/css/styles.css` exists before assuming styles are broken
- When adding new UI elements, use existing Tailwind classes from the color scheme
- The site is fully self-contained (no external APIs or databases)
- Mobile menu uses vanilla JavaScript (see bottom of `index.html`)
- All styling should be done via Tailwind classes (avoid custom CSS unless necessary)
